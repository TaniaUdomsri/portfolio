Product Tour

Design pattern, and shareable web component

We aim to build certain user experiences that very little help is needed. However, sometimes our users need some guidance. The Product Tour is a shareable web component thatprovides educational content on aspects of a particular page. These aspects can include new or changed features, light usage instructions, and step-by-step tours. Product Tour is a shareable web component on Amazon Seller Central. Amazon Seller Central is a web interface used by merchants or third-party sellers to sell their products directly to Amazon’s customers.

My responsibilities

Project lead, UX design, and Front-end engineer

Challenges

  1. To help, not to interrupt There are many contextual help components out there. We need to make sure that we design the right experience, with the right context, display at the right time, and not to intrude user experience.
  2. Consistent user experience This Product Tour feature would be used on different pages in Seller Central website. We need to make sure that Amazon seller would have a similar experience with the feature across the website. Also, we need to make the feature reusable so that different development teams can easily embed on their pages and speed up the development cycle.

Solutions

  1. Design process To make sure that we design the right experience, with the right context, display at the right time, and not to intrude user experience. We needed to:
    • Do a lot of research and competitive analysis to understand what is out there.
    • Collaborate and brainstorm with UX and user researchers from other teams across Amazon.
    • Create low to high fidelity prototype, receive feedback, and iterate.
    • Implement
    • Launch
    • Review usage metrics
  2. Reusable web component To speed up development cycle, I build a shareable web component that can be easily embedded on any pages in Seller Central website.

Outcomes and achievements

Because we need to make sure if we build a right experience, we tracked user usages across the tour experience. We first launched this on one page and collected the following metrics (1) % of users use the tour, and (2) % of users who completed all the tour’s steps. The results have been successful, we received positive feedback from users, and now this Product Tour is a design pattern and a shareable web component that is being used across Seller Central website.

Final implementation

Product Tour component was built using Bootstrap Tour. It is coded using JSP, CSS, and JavaScript on Java MVC framework.

Welcome modal

Product tour intro modal shows up when Sellers landed on the page. Sellers have a choice to view the tour or ignore it. By default, the selection is not to display the tour next time, and their preference will be stored.

Tour steps

Each tour step is displayed next to an element. Available positions are top/right/bottom/left. Users can close the tour at any time, and they always see how many steps of the tour. Image or animation can also be included in the content in .gif or html5 format.

Next steps

We are planning to work on other instructional help content use cases.
